# 系统架构

> 架构变更时由 AI 更新此文件。参考 Alice 方法论。

## 项目愿景

**构建一个类似 Alice 的人格化桌面 AI Agent。**

不只是一个工具，而是一个有性格、有记忆、能成长的数字伙伴：
- **人格化交互** — 有一致的性格特征和交流风格，不是冰冷的 Q&A 机器
- **持久记忆** — 记住用户的偏好、项目上下文、历史对话，越用越懂你
- **主动协作** — 不仅被动回答，还能主动提醒、建议、推进任务
- **本地优先** — 数据存储在用户本地，隐私可控

## 技术栈

| 层级 | 技术选择 |
|------|---------|
| 外壳 | Electron（主进程 Node.js + 渲染进程 Chromium） |
| 语言 | TypeScript 全栈，主进程与渲染进程共享类型定义 |
| 前端 | React + TailwindCSS + Lucide Icons |
| 存储 | SQLite（结构化）+ 向量数据库（语义检索）+ 本地文件系统 |
| 测试 | vitest / jest |
| 打包 | 待定（electron-builder / electron-forge） |

## 核心模块

### 1. Agent Loop（核心循环）

```
think → act → observe → think → ...
```

- 事件流使用 AsyncGenerator 模式
- 输出纯数据事件（AgentStreamEvent），不含 UI 逻辑

### 2. 工具系统

- 工具 vs 服务分离：AI 可见的是"工具"，AI 不可见的是"服务"
- 每个工具带声明式元数据：

```typescript
interface ToolMetadata {
  isReadOnly: boolean;
  isDestructive: boolean;
  isConcurrencySafe: boolean;
}
```

### 3. 记忆系统

参考 Alice 五层记忆体系（待实现时细化）：

| 层级 | 说明 |
|------|------|
| 系统提示 | 固定指令（角色 + 规则） |
| 长期记忆 | 向量库检索的历史上下文 |
| 项目记忆 | 当前项目的文件 + 决策记录 |
| 会话记忆 | 本轮对话 + 工具调用记录 |
| 临时上下文 | 当前正在处理的文件/代码片段 |

### 4. LLM 路由

- 统一适配器模式，支持多 Provider
- 模型切换只改配置，不改业务代码

### 5. 数据存储

- SQLite：对话、设置、工具调用记录等结构化数据
- 向量数据库：历史对话的语义索引
- 本地文件系统：项目记忆、用户画像

## IPC 通信

- 高频事件（文本 token）：批量合并后发送，防止渲染进程被淹没
- 结构性事件（工具开始/结束、完成、错误）：立即发送
- 权限确认提示：序列化处理，一次只弹一个
