# 规则反馈

> 开发过程中 AI 发现规则不合理/缺失/冲突时记录到此文件。
> 累计 3 条以上反馈时，AI 会主动建议用户审视并修订规则。

## 反馈格式

```
### #序号 - 简述

- **规则文件**：哪个 .mdc / .md
- **具体条款**：哪条规则
- **问题类型**：不合理 / 缺失 / 冲突 / 过于严格 / 过于宽松
- **场景描述**：在什么情况下触发的
- **建议**：怎么改
- **状态**：待审视 / 已采纳 / 已驳回
```

---

### #6 - Phase 11 文档更新规则在跨对话场景下未执行

- **规则文件**：`dev-workflow.mdc`
- **具体条款**：Phase 11 — 每次对话结束前按需更新文档
- **问题类型**：过于宽松（无强制机制，且跨对话容易丢失）
- **场景描述**：P1 到 P4 跨越多轮对话，每轮结束只更新了 `progress.md` 和 `features.md`，但 `architecture.md`、`changelog.md`、`api-contracts.md`、`decisions.md` 全部严重过时。原因分析：
  1. "按需更新"太模糊，没有 checklist 强制逐一检查
  2. 跨对话时，summary 恢复后容易忘记上一轮遗漏的文档更新
  3. 缺少"文档债务检测"机制
- **建议**：Phase 11 升级为 HARD-GATE，增加必查清单：每次对话结束前必须逐一检查 8 个核心文档是否需要更新，而不是"按需"
- **状态**：已采纳（2026-06-16，Phase 11 升级为 HARD-GATE + 8 项必查清单）
