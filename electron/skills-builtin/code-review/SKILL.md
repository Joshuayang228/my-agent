# 代码审查流程

## 1. 理解上下文
- 确认审查范围（哪些文件/函数）
- 了解代码的业务目标

## 2. 检查清单

### 正确性
- [ ] 逻辑是否正确，边界条件是否处理
- [ ] 错误处理是否完善
- [ ] 类型是否正确（TypeScript 场景）

### 安全性
- [ ] 无硬编码密钥/凭据
- [ ] 输入验证是否充分
- [ ] 无 SQL 注入 / XSS 风险

### 可维护性
- [ ] 函数职责是否单一
- [ ] 命名是否清晰
- [ ] 是否有不必要的复杂度

### 性能
- [ ] 有无 N+1 查询或不必要的循环
- [ ] 大数据量场景是否考虑

## 3. 输出格式

用以下级别分类反馈：
- 🔴 **必须修复**：有 Bug 或安全漏洞
- 🟡 **建议改进**：可以更好但不阻塞
- 🟢 **可选优化**：锦上添花
